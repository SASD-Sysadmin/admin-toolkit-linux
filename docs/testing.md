# Local Testing and Smoke Checks

This repository is currently designed to be validated locally. GitHub Actions examples may exist in `docs/ci-examples/`, but they are intentionally not required for day-to-day work.

## Basic validation before every commit

Run:

```bash
make check
```

This performs:

- Bash syntax checks for all `*.sh` files below `scripts/`
- executable-bit checks for shell scripts and non-script files
- `git diff --check` to detect whitespace problems

## Syntax-only check

```bash
make syntax
```

This is fast and useful after editing shell scripts.

## File mode check

```bash
make file-modes
```

Expected policy:

- shell scripts below `scripts/` should be executable
- documentation, examples, YAML files, `.gitignore`, `.gitattributes`, `LICENSE`, `Makefile` and Markdown files should not be executable

If file modes drift, repair them with:

```bash
find . -type f -not -path './.git/*' -exec chmod -x {} +
find scripts -type f -name '*.sh' -exec chmod +x {} +
```

## Read-only smoke test

Run:

```bash
make smoke
```

This creates a new local report directory below `reports/`.

The generated reports can contain hostnames, usernames, paths, package lists, service names and IP addresses. Do not commit real report output and do not share it publicly without review.

## Manual smoke test

```bash
./scripts/reporting/sasd-run-readonly-checks.sh --output ./reports/manual-test
head -80 ./reports/manual-test/INDEX.md
cat ./reports/manual-test/status.tsv
du -h ./reports/manual-test/*
```

## Expected non-zero statuses

Some audit-style scripts may return `1` when findings are present. This does not necessarily mean the script failed. The `INDEX.md` generated by the read-only collector explains this convention.

Exit status `2` or higher usually deserves closer review because it can indicate an argument problem, missing prerequisite or execution problem.

## WSL and container notes

Some Linux environments do not have all classic server components installed. For example, WSL or containers may not have `/etc/ssh/sshd_config` or a running OpenSSH server. The SSH scripts should report that as informational instead of failing hard.

## Before tagging a release

Suggested local checklist:

```bash
git status
make check
make smoke
git status --ignored -s | head -80
```

Confirm that generated report output is ignored by Git before tagging.
